# Project Structure

This document describes the current repository layout for Corporate AI Assistant.

## Root Overview

```text
ai-agent/
├── app.py                     # FastAPI application and HTTP routes
├── auth_kerberos.py          # Kerberos + LDAP authentication flow
├── auth_ldap.py              # Direct LDAP helper module
├── config.py                 # Central settings model
├── llm_gateway.py            # Queueing, leases, heartbeats, retries, capacity logic
├── runtime_healthcheck.py    # Container healthcheck entrypoint
├── scheduler.py              # Resource-aware scheduler
├── worker.py                 # Worker runtime and Ollama integration
├── docker-compose.yml        # Full deployment stack
├── Dockerfile                # Shared Python image for app/scheduler/workers
├── install.sh                # Production-oriented installer
├── start.sh                  # Compose-based start helper
├── clean.sh                  # Safe cleanup helper
├── templates/                # Jinja2 templates
├── static/                   # Frontend assets
├── deploy/                   # Nginx and Kerberos deployment assets
├── docs/                     # Additional operator documentation
└── README.md                 # Public project overview
```

## Runtime Layers

### API Plane

- `app.py`
- `templates/`
- `static/`

### Control Plane

- `llm_gateway.py`
- `redis` service in `docker-compose.yml`

### Scheduling Plane

- `scheduler.py`

### Inference Plane

- `worker.py`
- `ollama` service in `docker-compose.yml`

### Deployment and Operations

- `install.sh`
- `start.sh`
- `clean.sh`
- `deploy/nginx.conf`
- `deploy/setup-systemd.sh` and service files for legacy or custom operator workflows

## Generated Files Not Intended for Git

- `.env`
- `.install/`
- `deploy/certs/`
- `deploy/krb5.conf`
- `docker-compose.override.yml`
- `models/`
- Python cache directories

## Notes

- The public deployment path is Docker Compose based.
- Host-specific AD overrides are generated by `install.sh` into `docker-compose.override.yml` when needed.
- Local models are intentionally excluded from the repository and Docker build context.
