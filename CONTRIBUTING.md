# Руководство по внесению вклада

Спасибо за интерес к проекту Corporate AI Assistant! Мы приветствуем любой вклад.

## Как внести вклад

### Сообщение об ошибках

Если вы нашли ошибку:

1. Проверьте, не была ли она уже сообщена в Issues
2. Создайте новый Issue с подробным описанием:
   - Шаги для воспроизведения
   - Ожидаемое поведение
   - Фактическое поведение
   - Версия Python, ОС
   - Логи (если есть)

### Предложение новых функций

1. Создайте Issue с описанием функции
2. Объясните, зачем она нужна
3. Предложите возможную реализацию

### Pull Requests

1. Форкните репозиторий
2. Создайте ветку для вашей функции (`git checkout -b feature/amazing-feature`)
3. Внесите изменения
4. Убедитесь, что код соответствует стилю проекта
5. Добавьте тесты (если применимо)
6. Закоммитьте изменения (`git commit -m 'Add amazing feature'`)
7. Запушьте в ветку (`git push origin feature/amazing-feature`)
8. Откройте Pull Request

## Стиль кода

### Python

- Следуйте PEP 8
- Используйте type hints
- Документируйте функции docstrings
- Максимальная длина строки: 100 символов

Пример:
```python
def authenticate_user(username: str, password: str) -> Optional[Dict[str, Any]]:
    """
    Аутентификация пользователя через Kerberos.
    
    Args:
        username: Имя пользователя
        password: Пароль
        
    Returns:
        Словарь с данными пользователя или None
    """
    pass
```

### JavaScript

- Используйте современный ES6+ синтаксис
- Предпочитайте `const` и `let` вместо `var`
- Используйте async/await для асинхронного кода
- Добавляйте комментарии к сложной логике

### Коммиты

Используйте понятные сообщения коммитов:

- `feat: добавлена новая функция`
- `fix: исправлена ошибка в аутентификации`
- `docs: обновлена документация`
- `style: форматирование кода`
- `refactor: рефакторинг функции X`
- `test: добавлены тесты`
- `chore: обновлены зависимости`

## Тестирование

Перед отправкой PR убедитесь, что:

1. Код запускается без ошибок
2. Все существующие функции работают
3. Новый код протестирован
4. Нет конфликтов с main веткой

## Вопросы?

Если у вас есть вопросы, создайте Issue с меткой "question".

Спасибо за ваш вклад! 🎉
