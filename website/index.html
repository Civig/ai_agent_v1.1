Локальный AI внутри корпоративного контура

Corporate AI Assistant для безопасной работы с ИИ внутри инфраструктуры организации

Платформа спроектирована для внутреннего контура: локальный inference через Ollama, корпоративная аутентификация через Active Directory / Kerberos / LDAP, защищённый HTTPS ingress и управляемый запуск через Docker Compose. Главная цель — снизить риски передачи корпоративных данных и персональных данных во внешние облачные AI-сервисы.

Почему это важно Открыть каталог моделей
On-Prem модели и inference размещаются внутри корпоративной среды через Ollama
AD / Kerberos аутентификация и доступ основаны на корпоративной directory-инфраструктуре
Redis + Workers очереди, scheduler, worker pools и control plane для управляемого выполнения
Docker Compose основной поддерживаемый путь развёртывания на Linux VM
Это не публичный облачный AI-сервис. Это внутренняя корпоративная платформа, где оператор сам контролирует инфраструктуру, сертификаты, модели, доступ и эксплуатацию.

О продукте

Corporate AI Assistant — это корпоративный AI-ассистент для внутреннего контура, где локальный inference, внутренняя аутентификация и эксплуатационный контроль важны так же, как и сам чат-интерфейс.

🧠

Локальный inference через Ollama

Модели запускаются через локальный Ollama runtime. Каталог моделей в интерфейсе может заполняться из live-списка Ollama, а сами модели не хранятся в Git-репозитории.

🔐

Корпоративная аутентификация

Платформа рассчитана на Active Directory-среды: Kerberos ticket, LDAP/GSSAPI, membership в группах и выдача secure cookies для web-доступа.

⚙️

Управляемое выполнение запросов

Redis control plane, resource-aware scheduler и отдельные worker pools для chat, siem и batch позволяют отделить inference от web-слоя и сделать платформу более управляемой.

152-ФЗ и безопасность обработки данных

Ключевая идея платформы — держать обработку запросов, моделей и доступ пользователей внутри корпоративной среды, чтобы уменьшить риски, связанные с использованием внешних облачных AI-сервисов при работе с корпоративными данными и ПД.

Внутренний контур

On-Prem

Платформа ориентирована на внутреннее deployment: Linux VM, Docker Compose, локальный Ollama, внутренний Redis и Nginx как единственная публичная точка входа.

Контроль доступа

AD / LDAP

Аутентификация и доступ основаны на корпоративной directory-среде, а не на внешнем публичном аккаунте или стороннем облачном identity-provider по умолчанию.

Безопасный ingress

TLS / Nginx

Базовая схема эксплуатации держит приложение за Nginx, использует HTTPS ingress и публикует только 80/443, а Redis остаётся на внутренней Compose-сети.

Важно понимать

Compliance

Локальное развёртывание помогает снизить риски передачи данных наружу, но само по себе не заменяет организационные и правовые меры по защите персональных данных в организации.

Архитектура платформы

Основная архитектурная цепочка: Browser → Nginx → FastAPI → Kerberos/LDAP + Redis control plane → Scheduler → Worker pools → Ollama inference runtime.

Веб-слой

Browser + Nginx + FastAPI
Frontend / API
Пользователь работает через web UI. Внешний вход проходит через Nginx с TLS, а приложение работает в FastAPI.
HTTPS ingress Jinja2 FastAPI SSE streaming

Контроль и очереди

Redis + Scheduler
Control Plane
Redis используется для состояния очередей, heartbeat, lease/retry, rate limiting и session-state, а scheduler управляет admission control и назначением задач.
Redis queue state heartbeats scheduler

Inference-слой

Worker pools + Ollama
Inference Runtime
Worker pools разделяют chat, siem и batch-нагрузку, а inference выполняется через локальный Ollama runtime. Поддерживается CPU default и optional GPU profile.
worker-chat worker-siem worker-batch optional worker-gpu

Развёртывание и системные требования

Основной поддерживаемый путь — Linux VM с Docker Compose, доступом к AD / Kerberos / LDAP и локальным Ollama внутри stack.
